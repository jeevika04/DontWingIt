What is accessibility?

Accessibility is the practice of making your websites usable by as many people as possible.
- Mozilla Developer Network (https://developer.mozilla.org/en-US/docs/Learn/Accessibility/)

What is this?

DontWingIt was built with one goal in mind: to make accessible web content easier; So people don't wing it. DontWingIt tries to make development easier through gamification. We reward our users with points as they complete tasks.

Who should you be?

Just about anyone who wants to make an accessible website. While our target audience is anyone, our product works best with absolute beginners. Beginners are able to benefit from all our services we provide (learning component, gamification, development process guide, issue tracking).

Product format

Accessibility should be built in to your website. As such, you need to be thinking about accessibility every step of the development process. The development ideology our product outlines, and you will be following is design, develop, test.

Design

As the first phase of the development process, you can walk in with absolutely no ideas. Here, you will think about everything you want your product to have. Since this is not a product purely for accessibility, but for developing in an accessible manor, you will include both general features, and accessibility features. You can continue to add features throughout the entire development process. Features you add will show up in the development tab, and the testing tab.

Development

This is the second phase of the 3 phases of the development process. This is where the gamification component comes into play. Each feature you added in the design phase will show up here with a corresponding checkbox. As you finish developing a feature, check the box, and your score will increase.
For additional accessibility considerations as well as compliance, we have included the complete Web Content Accessibility Guidelines (WCAG 2.1) as items to complete. Here, you can audit your website as you are developing it, or soon after.

Testing

Testing is the last phase of the development process. It outlines both generic software testing as well as accessibility testing.
In the specific feature testing section, you can ensure your features function properly.

Accessibility Testing

Accessibility Testing follows 3 phases: Automated, Expert, User. It's important they're done in this order.

Automated Testing

There are several platforms developers can use to ensure their web content is accessible. Running your website through these will give you tons of insight. These are to be run before you conduct expert or user testing. This is because automated testing is much cheaper, and can be ran at your convenience. If this isn't run before expert testing, you will waste the expert's time with basic issues you could have found in a few seconds with automated tools.

Expert Testing

Expert testing is getting a professional to generate a report on the accessibility of your website. This should be done after automated testing, so the expert's time is spent finding issues automated tools are not capable of finding.
Our product allows you to submit all the issues the expert found and rank their severity as well. These issues will appear in the development phase.

User Testing

User testing is getting a differently-abled person to use your website. This should be done after last, so the user's unique perspective can be fully utilized, rather than finding issues experts, or worse, automated tools could find.
Our product allows you to submit all the issues the user found and rank their severity as well. These issues will appear in the development phase.
